> 说明：所有接口已做脱敏处理，隐藏了真实域名、用户ID等敏感信息，仅保留业务逻辑与安全分析价值。

## 一、项目背景
作为网络空间安全专业学生，对校园业务系统的接口进行系统性梳理与安全分析，是将理论知识落地到真实业务场景的重要实践。本项目收集并脱敏了校园系统的核心业务接口，用于Web安全学习、漏洞分析与合规性研究。

## 三、安全实践方向（网络空间安全专业）

### 1. Web安全核心漏洞验证
- **越权访问**：修改用户ID、分页参数，测试水平/垂直越权
- **未授权访问**：去掉Cookie/Token，直接请求接口验证权限绕过
- **SQL注入**：在查询条件、排序参数中注入特殊字符，测试注入风险
- **敏感信息泄露**：分析接口返回字段，检查是否泄露隐私数据
- **接口滥用**：重复提交申请、伪造审批状态，测试业务逻辑漏洞

### 2. 进阶研究方向
- 基于脱敏接口搭建模拟靶场，用于漏洞挖掘与利用练习
- 开发自动化安全扫描脚本，批量检测接口安全风险
- 结合《个人信息保护法》，分析校园系统数据合规性
