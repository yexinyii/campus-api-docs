# 校园系统接口安全靶场 - 漏洞测试手册

## 一、靶场环境说明

### 基础信息

- 靶场地址：`http://127.0.0.1:5000`
- 技术栈：Python + Flask
- 内置模块：统一认证、教务系统、学工系统
- 核心漏洞类型：未授权访问、水平 / 垂直越权、参数篡改、弱校验

## 二、漏洞测试与利用指南

### 1. 未授权访问漏洞（高危）

#### 漏洞接口

`GET /getLoginUser`

#### 漏洞原理

接口未做任何身份校验，直接返回当前登录用户信息，攻击者无需登录即可获取用户敏感数据。

#### 测试步骤

1. 直接在浏览器访问：`http://127.0.0.1:5000/getLoginUser`
2. 无需携带 Cookie/Token，即可获取用户 ID、姓名、学院等信息

#### 修复建议

- 所有接口添加全局身份校验中间件
- 基于 Session 或 Token 实现登录态校验，未登录请求直接返回 403

---

### 2. 水平越权漏洞（高危）

#### 漏洞接口

`POST /zdpz_cxZdpzList.html`

#### 漏洞原理

成绩查询接口仅校验参数格式，未校验当前用户与查询学生的权限关系，攻击者可通过修改`zd_fzdm`参数，查询任意学生的成绩数据。

#### 测试步骤

http

```
POST /zdpz_cxZdpzList.html
Content-Type: application/x-www-form-urlencoded

zd_fzdm=2023001
```

修改参数为其他学生 ID（如`2023002`），即可越权查看他人成绩。

#### 修复建议

- 后端校验请求用户 ID 与查询学生 ID 的一致性
- 禁止直接使用用户 ID 作为查询参数，改用会话绑定用户身份

---

### 3. 垂直越权漏洞（高危）

#### 漏洞接口

`GET /getUserPermissionRouters`

#### 漏洞原理

接口返回用户权限路由时，未根据用户角色过滤管理员路由，普通用户可直接获取后台管理接口路径，进一步尝试访问后台功能。

#### 测试步骤

1. 以普通用户身份请求：`http://127.0.0.1:5000/getUserPermissionRouters`
2. 响应中包含`/admin`等后台管理路由
3. 直接访问后台路由（如`/admin`），部分场景下可成功访问

#### 修复建议

- 后端根据用户角色动态过滤权限路由
- 后台接口添加角色权限校验，普通用户请求直接返回 403

---

### 4. 参数篡改漏洞（中危）

#### 漏洞接口

`POST /apply.do`

#### 漏洞原理

奖学金申请接口未校验提交参数的合法性，攻击者可篡改`amount`（申请金额）参数，修改申请的奖学金数额。

#### 测试步骤

http

```
POST /apply.do
Content-Type: application/json

{"amount":5000}
```

修改为`{"amount":10000}`即可篡改金额。

#### 修复建议

- 后端对提交参数进行范围校验，限制金额在合理区间
- 敏感业务参数不允许前端直接提交，由后端根据用户等级自动生成

---

### 5. 无 Cookie 身份校验漏洞（中危）

#### 漏洞接口

`GET /SYS_CARD_PERSONALDATA`

#### 漏洞原理

学生个人信息接口未校验请求 Cookie，攻击者无需登录即可直接访问任意学生的个人信息。

#### 测试步骤

直接访问：`http://127.0.0.1:5000/SYS_CARD_PERSONALDATA`

无需携带任何 Cookie，即可获取学生学号、姓名、专业等信息

#### 修复建议

- 所有用户相关接口添加 Cookie/Token 校验
- 关键接口增加请求频率限制，防止批量爬取数据
